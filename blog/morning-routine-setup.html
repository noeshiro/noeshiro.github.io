朝タスク自動生成ルーチン
セットアップガイド

2026.04.18

Claude Code Routines を使って、毎朝6:00にNotionタスクDBへ今日のタスクを自動投入する。朝Notionを開くだけで「今日やること」が揃っている状態を作る。

ルーチンが毎朝やること（3ステップ）

1繰り越しタスク検出

  • 対象: 優先度「本日中」かつステータス「未着手」のタスク
  • 処理: 今日の日付で同じ内容のタスクを新規作成
  • 入れ子構造: 元タスクと同じ親ページの下にぶら下げる
  • 引き継ぎ: タスク名、テーマ、内容をそのままコピー

2週1定常タスク

  • 曜日: 月曜日
  • タスク: 「note記事執筆」
  • 設定値: テーマ ["創作"] / 優先度 １週間以内
  • 配置: 「創作」親ページの下にぶら下げる

3マイルストーン分解

  • 対象: マイルストーンDBから期限が今日〜1ヶ月以内のもの
  • 処理: マイルストーンの内容を読み取り、具体的な作業タスクに分解して投入
  • 優先度の自動判定:
    • 期限まで7日以内 → 本日中
    • 期限まで14日以内 → １週間以内
    • 期限まで30日以内 → 優先度低
  • 配置: マイルストーンのテーマに対応する既存の親ページの下にぶら下げる
  • 重複防止: 同じマイルストーンから生成済みのタスクがあればスキップ

フィールド値の例

内容（テキスト）
ステータス未着手 / 完了
優先度本日中 / １週間以内 / 優先度低

テーマ'["創作"]' / '["その他"]' など（JSON配列文字列）

入れ子構造（実例）

その他 └ ワールドカップ ├ シカゴ→トロントとる ├ ダラス→シアトルとる └ チケットとったこと連絡 創作 └ 耳コピ ├ ドラム Aメロ ├ メロディ合わせる aメロ └ ベース最後のフィル合わせ…

セットアップ手順

前提条件

  • Claude Pro / Max / Team / Enterprise プラン
  • Claude Code on the web が有効
  • Notion MCP コネクターが接続済み

1. Routines画面を開く

claude.ai/code/routines にアクセス → 「New routine」をクリック

2. ルーチン設定

Name（名前）:

朝タスク自動生成

Prompt（プロンプト）:

あなたは毎朝実行されるタスク管理アシスタントです。
Notion MCP コネクターを使って、以下の3つの処理を順番に実行してください。

## 処理1: 繰り越しタスク検出

タスク管理DB（data_source_id: 30f55192-729f-80d6-b174-000b89cd078d）を検索し、
以下の条件に合うタスクを見つけてください：
- ステータスが「未着手」
- 優先度が「本日中」
- 期限が今日より前（期限フィールドが空のものは対象外）

見つかった各タスクについて：
1. 同じ親ページの下に新しいタスクを作成
2. タスク名・内容・テーマをそのままコピー
3. 優先度を「本日中」、期限を今日の日付に設定
4. ステータスを「未着手」に設定

## 処理2: 週1定常タスク

今日が月曜日の場合のみ実行：
- 「創作」カテゴリの親ページを検索
- その下に以下のタスクを作成：
  - タスク名: 「note記事執筆」
  - テーマ: ["創作"]
  - 優先度: 「１週間以内」
  - ステータス: 「未着手」

月曜日でなければこの処理はスキップ。

## 処理3: マイルストーン分解

マイルストーンDB（data_source_id: c3873932-5536-49f6-8bce-d87fad3a7adb）を検索し、
期限が今日から30日以内のマイルストーンを取得。

各マイルストーンについて：
1. 内容を読み取り、具体的な作業タスク（2〜5個程度）に分解
2. 優先度を期限から自動判定：
   - 期限まで7日以内 → 「本日中」
   - 期限まで14日以内 → 「１週間以内」
   - 期限まで30日以内 → 「優先度低」
3. タスク管理DBの対応するテーマの親ページの下にタスクを作成
4. 既に同じタスク名が存在する場合はスキップ（重複防止）

## 実行後の出力

処理結果をまとめて報告してください：
- 繰り越したタスク数と内容
- 週1タスクの作成有無
- マイルストーンから分解・作成したタスク数と内容
- スキップした項目（重複など）

3. リポジトリ

リポジトリは不要（Notionコネクター経由のみで動作）。リポジトリ選択は空のままでOK。

4. トリガー設定

  • 「Add trigger」→「Schedule」を選択
  • 頻度: Daily
  • 時刻: 6:00 AM（JST）

5. コネクター

  • Notion が含まれていることを確認
  • 不要なコネクター（Gmail、Google Drive等）は外してOK

6. 「Create」をクリック

動作確認

  1. ルーチン詳細ページで「Run now」をクリックして手動実行
  2. セッションが開くので、実行内容を確認
  3. Notionのタスク管理DBを開いて、タスクが正しく作成されているか確認

チェックポイント

  • 繰り越しタスクが今日の日付で作成されている
  • 入れ子構造（親ページ）が正しく維持されている
  • 月曜日に「note記事執筆」が作成される
  • マイルストーンから分解されたタスクの優先度が正しい
  • 重複タスクが作成されていない

カスタマイズメモ

週1タスクを追加したい場合

プロンプトの「処理2」に曜日とタスク内容を追記する。

例: 水曜日に「耳コピ進捗確認」を追加

今日が水曜日の場合：
- 「創作」カテゴリ → 「耳コピ」の下に
- タスク名: 「耳コピ進捗確認」
- 優先度: 「１週間以内」

実行時刻を変えたい場合

Routines画面 → ルーチンを編集 → トリガーの時刻を変更

日常タスクを追加したい場合

プロンプトに「処理4」として追記すればOK。例: 毎日「家計簿つける」を入れる、など。
