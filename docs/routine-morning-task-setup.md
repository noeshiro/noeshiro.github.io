# 朝タスク自動生成ルーチン — セットアップガイド

## 概要

Claude Code Routines を使って、毎朝6:00にNotionタスクDBへ今日のタスクを自動投入する。
朝Notionを開くだけで「今日やること」が揃っている状態を作る。

---

## ルーチンが毎朝やること（3ステップ）

### Step 1: 繰り越しタスク検出

- **対象**: 優先度「本日中」かつステータス「未着手」で期限が過ぎたタスク
- **処理**: 今日の日付で同じ内容のタスクを新規作成
- **入れ子構造**: 元タスクと同じ「親アイテム」リレーションを設定
- **引き継ぎ**: タスク名、テーマ、内容をそのままコピー

### Step 2: 週1定常タスク

- **曜日**: 月曜日
- **タスク**: 「note記事執筆」
- **設定値**: テーマ `創作` / 優先度 `１週間以内`
- **配置**: 「創作」親ページの下にぶら下げる（親アイテムリレーション設定）

### Step 3: マイルストーン分解

- **対象**: マイルストーンDB（マイルストーン 2026）から期限が今日〜1ヶ月以内のもの
- **処理**: マイルストーンの「メモ」を読み取り、具体的な作業タスクに分解して投入
- **優先度の自動判定**:
  - 期限まで7日以内 → `本日中`
  - 期限まで14日以内 → `１週間以内`
  - 期限まで30日以内 → `優先度低`
- **配置**: カテゴリに対応するテーマの親ページの下にぶら下げる
- **重複防止**: 同じマイルストーンから生成済みのタスクがあればスキップ

## Notion DB 情報

| DB | data_source_id | 正式名称 |
|---|---|---|
| タスク管理DB | `30f55192-729f-80d6-b174-000b89cd078d` | タスク管理DB |
| マイルストーンDB | `c3873932-5536-49f6-8bce-d87fad3a7adb` | マイルストーン 2026 |

### タスク管理DBのフィールド

| フィールド | 型 | 値の例 |
|---|---|---|
| タスク名 | title | `シカゴ→トロントとる` |
| 内容 | text | （テキスト） |
| ステータス | status | `まち` / `未着手` / `進行中` / `完了` |
| 優先度 | select | `本日中` / `１週間以内` / `優先度低` / `１か月以内` |
| 期限 | date | `2026-04-13` |
| テーマ | multi_select | `恋愛` / `創作` / `仕事` |
| 親アイテム | relation (self) | 親タスクへのリレーション（1件のみ） |
| サブアイテム | relation (self) | 子タスクへのリレーション（複数可） |

### マイルストーンDBのフィールド

| フィールド | 型 | 値の例 |
|---|---|---|
| タスク | title | マイルストーン名 |
| メモ | text | 詳細内容 |
| ステータス | select | `未着手` / `進行中` / `完了` |
| カテゴリ | select | `制作` / `発信` / `収益化` / `スキル` |
| Q | select | `Q2` / `Q3` / `Q4` |
| 月 | select | `4月` 〜 `12月` |
| 期限 | date | `2026-06-30` |

### カテゴリ → テーマ マッピング

マイルストーンDBの「カテゴリ」とタスク管理DBの「テーマ」は値が異なるため、
以下のマッピングでタスクを作成する：

| マイルストーン カテゴリ | タスク管理 テーマ |
|---|---|
| 制作 | 創作 |
| 発信 | 創作 |
| 収益化 | 仕事 |
| スキル | 仕事 |

> **注意**: このマッピングは仮定です。実際の運用に合わせて調整してください。

### 入れ子構造（実例）

```
その他（親タスク）
  └ ワールドカップ（子タスク、親アイテム=その他）
      ├ シカゴ→トロントとる（親アイテム=ワールドカップ）
      ├ ダラス→シアトルとる（親アイテム=ワールドカップ）
      └ チケットとったこと連絡（親アイテム=ワールドカップ）
創作（親タスク）
  └ 耳コピ（子タスク、親アイテム=創作）
      ├ ドラム Aメロ（親アイテム=耳コピ）
      ├ メロディ合わせる aメロ（親アイテム=耳コピ）
      └ ベース最後のフィル合わせ…（親アイテム=耳コピ）
```

## セットアップ手順

### 前提条件

- Claude Pro / Max / Team / Enterprise プラン
- Claude Code on the web が有効
- Notion MCP コネクターが接続済み

### 手順

#### 1. Routines画面を開く

`https://claude.ai/code/routines` にアクセス → 「New routine」をクリック

#### 2. ルーチン設定

**Name（名前）:**
```
朝タスク自動生成
```

**Prompt（プロンプト）:**

```
あなたは毎朝実行されるタスク管理アシスタントです。
Notion MCP コネクターを使って、以下の3つの処理を順番に実行してください。
今日の日付を確認してから処理を開始してください。

## 処理1: 繰り越しタスク検出

タスク管理DB（data_source_id: 30f55192-729f-80d6-b174-000b89cd078d）を
query_data_sources で検索し、以下の条件に合うタスクを見つけてください：
- ステータスが「未着手」
- 優先度が「本日中」
- 期限が今日より前（期限フィールドが空のものは対象外）

見つかった各タスクについて：
1. 元タスクの「親アイテム」リレーションを確認
2. 同じ「親アイテム」を持つ新しいタスクを作成（create-pages で data_source_id 指定）
3. タスク名・内容・テーマをそのままコピー
4. 優先度を「本日中」、期限を今日の日付に設定
5. ステータスを「未着手」に設定
6. 「親アイテム」に元タスクと同じ親ページのURLを設定

## 処理2: 週1定常タスク

今日が月曜日の場合のみ実行：
1. タスク管理DBで「創作」テーマを持つ親ページ的なタスクを検索
2. その下に以下のタスクを作成：
   - タスク名: 「note記事執筆」
   - テーマ: 創作（multi_select）
   - 優先度: 「１週間以内」
   - ステータス: 「未着手」
   - 親アイテム: 見つけた「創作」親ページのURL

月曜日でなければこの処理はスキップ。

## 処理3: マイルストーン分解

マイルストーンDB（data_source_id: c3873932-5536-49f6-8bce-d87fad3a7adb）を
query_data_sources で検索し、期限が今日から30日以内のマイルストーンを取得。
ステータスが「完了」のものは除外。

各マイルストーンについて：
1. 「メモ」フィールドを読み取り、具体的な作業タスク（2〜5個程度）に分解
2. 優先度を期限から自動判定：
   - 期限まで7日以内 → 「本日中」
   - 期限まで14日以内 → 「１週間以内」
   - 期限まで30日以内 → 「優先度低」
3. カテゴリからテーマへのマッピング：
   - 制作 → 創作
   - 発信 → 創作
   - 収益化 → 仕事
   - スキル → 仕事
4. タスク管理DBで対応するテーマの親ページを検索し、その下にタスクを作成
   （親アイテムリレーションを設定）
5. 既に同じタスク名が存在する場合はスキップ（重複防止）

## 実行後の出力

処理結果をまとめて報告してください：
- 繰り越したタスク数と内容
- 週1タスクの作成有無
- マイルストーンから分解・作成したタスク数と内容
- スキップした項目（重複など）
```

#### 3. リポジトリ

リポジトリは不要（Notionコネクター経由のみで動作）。
リポジトリ選択は空のままでOK。

#### 4. トリガー設定

- 「Add trigger」→「Schedule」を選択
- 頻度: **Daily**
- 時刻: **6:00 AM**（JST）

#### 5. コネクター

- **Notion** が含まれていることを確認
- 不要なコネクター（Gmail、Google Drive等）は外してOK

#### 6. 「Create」をクリック

## 動作確認

1. ルーチン詳細ページで「Run now」をクリックして手動実行
2. セッションが開くので、実行内容を確認
3. Notionのタスク管理DBを開いて、タスクが正しく作成されているか確認

### チェックポイント

- [ ] 繰り越しタスクが今日の日付で作成されている
- [ ] 親アイテムリレーションが正しく設定されている（入れ子構造維持）
- [ ] 月曜日に「note記事執筆」が作成される
- [ ] マイルストーンから分解されたタスクの優先度が正しい
- [ ] カテゴリ→テーマのマッピングが正しい
- [ ] 重複タスクが作成されていない

## カスタマイズメモ

### カテゴリ→テーマのマッピングを変更したい場合

プロンプトの「処理3」のマッピング表を書き換える。

### 週1タスクを追加したい場合

プロンプトの「処理2」に曜日とタスク内容を追記する。

例: 水曜日に「耳コピ進捗確認」を追加
```
今日が水曜日の場合：
- 「創作」テーマの親ページ → 「耳コピ」の下に
- タスク名: 「耳コピ進捗確認」
- 優先度: 「１週間以内」
- 親アイテム: 「耳コピ」ページのURL
```

### 実行時刻を変えたい場合

Routines画面 → ルーチンを編集 → トリガーの時刻を変更

### 日常タスクを追加したい場合

プロンプトに「処理4」として追記すればOK。
例: 毎日「家計簿つける」を入れる、など。

### テーマの選択肢を増やしたい場合

タスク管理DBのテーマ multi_select に選択肢を追加し、
マイルストーンDBのカテゴリとのマッピングもプロンプトに追記する。
