# 朝タスク自動生成ルーチン — セットアップガイド

## 概要

Claude Code Routines を使って、毎朝6:00にNotionタスクDBへ今日のタスクを自動投入する。
朝Notionを開くだけで「今日やること」が揃っている状態を作る。

---

## ルーチンが毎朝やること（3ステップ）

### Step 1: 繰り越しタスク検出

- **対象**: 優先度「本日中」かつステータス「未着手」のタスク
- **処理**: 今日の日付で同じ内容のタスクを新規作成
- **入れ子構造**: 元タスクと同じ親ページの下にぶら下げる
- **引き継ぎ**: タスク名、テーマ、内容をそのままコピー

### Step 2: 週1定常タスク

- **曜日**: 月曜日
- **タスク**: 「note記事執筆」
- **設定値**: テーマ `["創作"]` / 優先度 `１週間以内`
- **配置**: 「創作」親ページ（ID: `34055192-729f-8023-8584-c1c9acbc77c4`）の下にぶら下げる

### Step 3: マイルストーン分解

- **対象**: マイルストーンDB（「マイルストーン 2026」）から期限が今日〜1ヶ月以内のもの
- **処理**: マイルストーンのメモを読み取り、具体的な作業タスクに分解して投入
- **優先度の自動判定**:
  - 期限まで7日以内 → `本日中`
  - 期限まで14日以内 → `１週間以内`
  - 期限まで30日以内 → `優先度低`
- **配置**: マイルストーンのカテゴリに対応する既存の親ページの下にぶら下げる
- **重複防止**: 同じマイルストーンから生成済みのタスクがあればスキップ

| マイルストーン カテゴリ | → タスク管理 テーマ | 親ページ |

| 制作 | 創作 | 創作 (`34055192-729f-8023-8584-c1c9acbc77c4`) |
| 発信 | 創作 | 創作 (`34055192-729f-8023-8584-c1c9acbc77c4`) |
| 収益化 | 仕事 | その他 (`34055192-729f-8036-882b-d9d685859d2a`) |

## 共通情報

- タスク管理DB data_source_id: 30f55192-729f-80d6-b174-000b89cd078d
- マイルストーンDB data_source_id: c3873932-5536-49f6-8bce-d87fad3a7adb
- 親ページID:
  - 創作: 34055192-729f-8023-8584-c1c9acbc77c4
  - その他: 34055192-729f-8036-882b-d9d685859d2a
- マイルストーン「カテゴリ」→ タスク「テーマ」変換:
  - 制作 → テーマ「創作」、親ページ「創作」
  - 発信 → テーマ「創作」、親ページ「創作」
  - 収益化 → テーマ「仕事」、親ページ「その他」
  - スキル → テーマ「創作」、親ページ「創作」

## 処理1: 繰り越しタスク検出

タスク管理DBを検索し、以下の条件に合うタスクを見つけてください：
- ステータスが「未着手」
- 優先度が「本日中」
- 期限が今日より前（期限フィールドが空のものは対象外）

見つかった各タスクについて：
1. 元タスクの「親アイテム」リレーションを確認し、同じ親ページの下に新しいタスクを作成
2. タスク名・内容・テーマをそのままコピー
3. 優先度を「本日中」、期限を今日の日付に設定（date:期限:start を使用）
4. ステータスを「未着手」に設定

## 処理2: 週1定常タスク

今日が月曜日の場合のみ実行：
- 親ページ「創作」（ID: 34055192-729f-8023-8584-c1c9acbc77c4）の下にタスクを作成
- 作成するタスク:
  - タスク名: 「note記事執筆」
  - テーマ: ["創作"]
  - 優先度: 「１週間以内」
  - ステータス: 「未着手」
- 同じ週に既に「note記事執筆」が存在する場合はスキップ

月曜日でなければこの処理はスキップ。

## 処理3: マイルストーン分解

マイルストーンDB（data_source_id: c3873932-5536-49f6-8bce-d87fad3a7adb）を検索し、
以下の条件に合うマイルストーンを取得：
- ステータスが「完了」以外
- 期限（date:期限:start）が今日から30日以内

各マイルストーンについて：
1. 「メモ」フィールドの内容を読み取り、具体的な作業タスク（2〜5個程度）に分解
2. 優先度を期限から自動判定：
   - 期限まで7日以内 → 「本日中」
   - 期限まで14日以内 → 「１週間以内」
   - 期限まで30日以内 → 「優先度低」
3. カテゴリ → テーマ変換（上記共通情報参照）を適用
4. 対応する親ページの下にタスクを作成（親アイテムリレーションを設定）
5. タスク管理DBに同じタスク名が既に存在する場合はスキップ（重複防止）

- スキップした項目（重複など）

## DB検証で見つかった差異（2026-05-22確認）

オリジナルガイドと実際のDBスキーマの間に以下の差異があった。本ドキュメントは修正済み。

| 項目 | オリジナル | 実際 |
|---|---|---|
| テーマの選択肢 | 「その他」を含む | 「恋愛」「創作」「仕事」の3つのみ |
| 優先度の選択肢 | 3段階 | 4段階（「１か月以内」も存在） |
| ステータスの選択肢 | 2つ | 4つ（「まち」「進行中」も存在） |
| マイルストーンDB名 | マイルストーンDB | マイルストーン 2026 |
| マイルストーンのタイトル列 | 記載なし | 「タスク」（「タスク名」ではない） |
| マイルストーンの内容列 | 記載なし | 「メモ」（「内容」ではない） |
| マイルストーンのカテゴリ | テーマと同じ前提 | 独自の選択肢（制作/発信/収益化/スキル） |
| 「仕事」「恋愛」親ページ | 存在する前提 | 現時点では未作成（「創作」「その他」のみ） |

## カスタマイズメモ

### 週1タスクを追加したい場合

プロンプトの「処理2」に曜日とタスク内容を追記する。

例: 水曜日に「耳コピ進捗確認」を追加
```
今日が水曜日の場合：
- 親ページ「創作」の下に
- タスク名: 「耳コピ進捗確認」
- 優先度: 「１週間以内」
```

### 実行時刻を変えたい場合

Routines画面 → ルーチンを編集 → トリガーの時刻を変更

### 日常タスクを追加したい場合

プロンプトに「処理4」として追記すればOK。
例: 毎日「家計簿つける」を入れる、など。

### 「仕事」「恋愛」用の親ページを追加したい場合

タスク管理DBにトップレベルのページとして「仕事」「恋愛」を作成し、
プロンプトの「共通情報」セクションにIDを追加する。
